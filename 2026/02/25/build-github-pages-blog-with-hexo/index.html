从零搭建 GitHub Pages 个人博客：Hexo + Next 主题完全指南

发表于 2026-02-25 分类于 技术教程
详细介绍如何使用 Hexo 框架和 Next 主题搭建托管在 GitHub Pages 上的个人博客，包含完整的环境配置、主题安装和高级功能设置。

你是否曾有过这样的想法——想拥有一个属于自己的技术博客，却苦于没有服务器、不会维护数据库？如果你有 WordPress 博客的搭建经验，可能会被服务器费用、域名备案、安全维护等问题困扰。今天我要介绍的方法，可以让你零成本拥有一个完全属于你的个人博客，而且只需要几分钟就能上线。

为什么选择 GitHub Pages + Hexo？

在开始之前，让我们先搞清楚为什么这个方案值得尝试。

GitHub Pages 是 GitHub 提供的免费静态网站托管服务，每个账户可以托管一个名为 用户名.github.io 的仓库，访问地址就是 https://用户名.github.io。更重要的是，它是完全免费的，而且自带 CDN 加速。

Hexo 是一个快速、简洁且高效的博客框架，它的特点是：

  • 静态生成——所有页面都是 HTML 文件，无需服务器
  • Markdown 支持——用最熟悉的语法写博客
  • 丰富主题——社区提供了大量精美主题
  • 插件生态——支持 RSS、搜索、评论等各种功能

这两者的结合，意味着你只需要写 Markdown，就能得到一个高性能、高颜值的技术博客。

环境准备

在开始之前，你需要准备以下环境：

安装 Node.js

Hexo 基于 Node.js 运行，所以第一步是安装 Node.js。访问 Node.js 官网，下载 LTS（长期支持）版本，直接运行安装程序即可。

安装 Git

Git 用于将博客代码推送到 GitHub。Windows 用户建议安装 Git for Windows，安装时选择 “Use Git from Git Bash only”。

创建 GitHub 仓库

登录 GitHub，点击右上角的 “+” → “New repository”，创建一个名为 你的用户名.github.io 的公开仓库。例如我的用户名是 Jackisome，就创建 Jackisome.github.io。

💡 小贴士：仓库名必须是 用户名.github.io 的格式，GitHub 才会自动将其识别为 Pages 站点。

1. 安装 Utterances App

  1. 访问 https://github.com/apps/utterances
  2. 点击 “Install”
  3. 选择 “Only select repositories”
  4. 选中你的博客仓库（如 Jackisome.github.io）
  5. 点击 “Install”
